## Reporting a Vulnerability

We take the security of Dakiya very seriously. If you discover a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

**🚨 Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please report them privately using one of the following methods:

### How to Report

1. **Email:** Send a detailed email to `[your-email@example.com]` (Please replace this with the actual security contact email).
2. **GitHub Private Vulnerability Reporting:** If enabled on this repository, navigate to the **Security** tab, click **Advisories**, and then click **Report a vulnerability**.

### What to Include

Please provide as much information as possible to help us reproduce and resolve the issue:
*   A description of the vulnerability and its potential impact.
*   Step-by-step instructions to reproduce the issue.
*   Any proof-of-concept (PoC) code or scripts.
*   Suggestions for a mitigation or fix, if you have any.

### Response Process

*   We will acknowledge receipt of your vulnerability report within **48 hours**.
*   We will investigate the issue and send you updates on our progress.
*   Once the vulnerability is patched, we will coordinate with you to publish a security advisory and notify users to update.

We ask that you please keep the details of the vulnerability confidential until a fix has been released and announced to protect the users of this project. Thank you for your contribution to keeping Dakiya secure!
